PROJECT REPORT: GU HEALTH MIND

AI-Powered Mental Health Prediction Application
1. ABSTRACT
GU Health Mind is a modern Android application designed to provide preliminary mental health screenings using Artificial Intelligence. By leveraging Natural Language Processing (NLP) through a remote web API, the application analyzes user-provided text input to predict potential mental health issues like Anxiety, Depression, Stress, Bipolar Disorder, or Suicidal tendencies. The app is built using industry-standard Clean Architecture and MVI (Model-View-Intent) patterns to ensure scalability and maintainability.
2. OBJECTIVES
•
To create an intuitive interface for users to describe their emotional state.
•
To integrate a remote AI model via REST API for real-time analysis.
•
To implement a robust filtering algorithm that identifies only high-risk categories (Class 1) and selects the most probable issue.
•
To provide a premium user experience through modern Jetpack Compose animations and "Glassmorphism" UI design.
3. SYSTEM ARCHITECTURE
The project follows Clean Architecture principles, divided into three distinct layers:
3.1 Domain Layer (Business Logic)
•
Entities: Pure Kotlin data classes (e.g., PredictionResult) representing the core business models.
•
Use Cases: Contains the "GetPredictionUseCase," which encapsulates the business logic of filtering API responses to find the maximum probability among positive detections.
3.2 Data Layer (Infrastructure)
•
Repository Pattern: Implementation of the domain repository.
•
Remote Source: Uses Ktor Client for asynchronous network requests.
•
DTOs: Data Transfer Objects used for JSON parsing via Kotlinx Serialization.
3.3 Presentation Layer (UI)
•
MVI Pattern:
◦
State: A single source of truth for the UI.
◦
Intent/Event: User actions sent to the ViewModel.
◦
Effect: Side effects like navigation or error messages.
•
Jetpack Compose: Declarative UI toolkit for building a reactive interface.

5. TECHNICAL IMPLEMENTATION DETAILS
5.1 AI Analysis Logic
The application processes a complex JSON response from the remote AI model. The technical requirement for filtering is implemented as follows:
1.
Filtering: The app iterates through all predicted categories (Anxiety, Depression, etc.).
2.
Class Verification: It only considers categories where the AI indicates class: 1 (Positive detection).
3.
Max Probability: If multiple categories are detected, the app uses a comparison algorithm to select the category with the highest numerical probability.
5.2 Networking with Ktor
Ktor was selected over Retrofit for its native support for Coroutines and lightweight footprint. It handles the POST request to the HuggingFace-hosted model:
Kotlin
val response = client.post("API_URL") {
    contentType(ContentType.Application.Json)
    setBody(PredictionRequest(text))
}.body<PredictionResponseDto>()
5.3 UI Enhancements & Animations
•
Animated Gradient: Implemented using infiniteRepeatable and Brush.linearGradient to create a moving background that shifts between deep violet and royal blue.
•
Glassmorphism: UI cards use semi-transparent white backgrounds (alpha = 0.2f) combined with rounded corners to mimic the "frosted glass" look found in modern OS designs.
•
State-Aware UI: The analysis button dynamically transforms into a loading indicator during network operations using AnimatedContent.

Endpoint: https://deepaksinghrx-menhel.hf.space/predict

Method: POST
•
Request Format: JSON { "text": "user input string" }
•
