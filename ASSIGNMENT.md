# Advanced Analytics Assignment

## Overview
This assignment is based on the Lab 3 Advanced Analytics project. Students will use the generated e-commerce dataset and the analytics pipeline to perform a full exploratory data analysis, identify data quality issues, detect outliers, and produce actionable business insights.

## Goals
- Apply advanced NumPy and Pandas techniques
- Perform data quality assessment and profiling
- Detect outliers using multiple statistical methods
- Analyze correlations and segment performance
- Produce a written report with insights and recommendations

## Dataset
The dataset is generated by `Lab3_Advanced_Analytics/src/data_generator.py` and saved to:
- `Lab3_Advanced_Analytics/data/ecommerce_transactions.csv`

It contains realistic e-commerce transaction records with columns such as:
- `transaction_id`
- `customer_id`
- `timestamp`
- `product_category`
- `product_price`
- `quantity`
- `discount_pct`
- `customer_age`
- `is_returning`
- `device_type`
- `region`
- `payment_method`
- `total_price`
- `final_price`

## Assignment Tasks

### Task 1: Data Quality Assessment
1. Load the dataset from `ecommerce_transactions.csv`.
2. Calculate missing values for each column and the overall missing percentage.
3. Identify duplicate rows and duplicate transaction groups.
4. Report data types and memory usage.

### Task 2: Advanced Statistical Profiling
1. Compute descriptive statistics for numeric columns.
   - count, mean, std, min, 25th percentile, median, 75th percentile, max
2. Calculate skewness and kurtosis for each numeric column.
3. Run a normality test on at least three numeric variables and interpret the results.

### Task 3: Outlier Detection
1. Detect outliers using the IQR method for numeric columns.
2. Detect outliers with the Z-score method for numeric columns.
3. Compare and summarize the differences between the two methods.
4. List the top 10 transactions flagged as outliers in `final_price`.

### Task 4: Correlation and Relationship Analysis
1. Compute Pearson and Spearman correlation matrices for numeric columns.
2. Identify at least three strong relationships (|r| > 0.3).
3. Interpret the relationships and explain whether they are meaningful for business decisions.
4. Use at least one visualization technique to illustrate a key relationship.

### Task 5: Segment Analysis
1. Analyze revenue and average order value by `product_category`.
2. Analyze revenue by `region` and `device_type`.
3. Calculate the proportion of revenue from returning customers versus new customers.
4. Create a customer segmentation table using quartiles of `final_price` or total spending.

### Task 6: Time Series and Temporal Patterns
1. Aggregate daily revenue and compute 7-day and 14-day moving averages.
2. Analyze revenue by day of week and identify the busiest days.
3. Identify any seasonal or periodic patterns in the dataset.
4. Create one visualization showing a temporal trend.

### Task 7: Business Insights and Recommendations
1. Summarize the top 5 findings from your analysis.
2. Provide at least 3 actionable business recommendations.
3. Highlight any data quality issues that should be addressed before production.
4. Suggest one additional feature or dataset that would improve future analysis.

## Extra Credit
- Create a new feature engineering pipeline and add at least two derived columns.
- Use hierarchical indexing (`MultiIndex`) to compare weekly sales by product category.
- Build a simple scoring model using the available features to identify high-value customers.
- Implement a logistic regression model to predict whether a transaction is from a returning customer.

## Deliverables
1. A written report in Markdown or PDF describing your process, findings, and recommendations.
2. A Python script or notebook that performs the full analysis.
3. Output files generated by the analysis, such as:
   - `data_quality_report.csv`
   - `statistical_profile.csv`
   - `outlier_detection.csv`
   - `strong_correlations.csv`
   - `segment_analysis.csv`
4. Visualizations embedded in the notebook or saved as image files.
